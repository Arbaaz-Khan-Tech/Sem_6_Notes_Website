2. Functional vs. Non-Functional Requirements

Functional Requirements

What the system does (features)

Example: "User can reset password via email"

Non-Functional Requirements

How well the system performs (quality attributes)

Example: "System loads in <2s (performance)"

Aspect Functional Non-Functional
Focus Features (WHAT) Quality (HOW WELL)
Testing Unit/Integration Tests Performance/Security Tests
Priority Core functionality User experience/scalability

Mnemonic: "Functional = Features (WHAT), Non-Functional = Numbers (HOW FAST/SECURE)!"
