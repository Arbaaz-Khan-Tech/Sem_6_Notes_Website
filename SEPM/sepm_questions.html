Software Engineering Fundamentals

Definition & Umbrella Activities

  • Software Engineering: Systematic approach to software development
  • Umbrella Activities:
    • Project Tracking (monitoring progress)
    • Risk Management (identify/mitigate risks)
    • Quality Assurance (SQA, testing)
    • Configuration Management (SCM)
    • Documentation

4'Ps of Project Management

  • People (team, stakeholders)
  • Product (requirements, scope)
  • Process (SDLC models)
  • Project (scheduling, resources)

Memory Shortcut: "PPPP" (People, Product, Process, Project)

2

Software Development Life Cycle

Model Key Features When to Use
Waterfall Linear, sequential phases Stable requirements
Agile Iterative, customer feedback Dynamic requirements
Incremental Deliver in modules Large projects
Spiral Risk-driven, combines Waterfall + Agile High-risk projects

Agile vs. Traditional

  • Agile: Flexible, sprints, Scrum/Kanban
  • Traditional (Waterfall): Rigid, phase-wise

Memory Shortcut: "Agile = Flexible, Waterfall = Fixed"

3

Requirements Engineering

Functional vs. Non-Functional

Functional

What the system does (e.g., login)

Non-Functional

How the system performs (e.g., speed, security)

SRS Structure

  1. Introduction (purpose, scope)
  2. Functional Requirements (use cases)
  3. Non-Functional Requirements (performance, security)
  4. User Interface (mockups)

Project Management

COCOMO Model

Effort = a × (KLOC)ᵇ

  • Organic (a=2.4, b=1.05)
  • Semi-Detached (a=3.0, b=1.12)
  • Embedded (a=3.6, b=1.20)

CPM & PERT

Expected Time = (O + 4M + P)/6

  • O: Optimistic
  • M: Most likely
  • P: Pessimistic
