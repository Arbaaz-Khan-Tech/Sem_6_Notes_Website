Design Phase

Quality Assurance focuses on design patterns and architecture reviews.

Implementation Phase

Configuration Management tracks code changes and versions.

Testing Phase

Quality Assurance is most intensive during testing.

Deployment Phase

Risk Management focuses on deployment strategies and rollback plans.

Maintenance Phase

Process Improvement analyzes what worked and what didn't.

Functional vs. Non-Functional Requirements

Understanding the key differences between what a system does and how well it performs:

Functional Requirements

"What the system DOES"

Define specific actions, inputs, and outputs that the system must perform.

Industry-Specific Examples

Healthcare

Functional Requirements
  • Patient record management
  • Appointment scheduling
  • Prescription ordering
  • Insurance claim processing
Non-Functional Requirements
  • HIPAA compliance
  • 99.99% uptime for critical systems
  • Data encryption at rest and in transit
  • Audit trail for all data access

E-commerce

Functional Requirements
  • Product catalog browsing
  • Shopping cart management
  • Checkout process
  • Order tracking
Non-Functional Requirements
  • Page load time under 3 seconds
  • Support for 10,000+ concurrent users
  • Mobile-responsive design
  • PCI DSS compliance for payments

Finance

Functional Requirements
  • Account balance checking
  • Fund transfers between accounts
  • Bill payments
  • Transaction history
Non-Functional Requirements
  • Real-time transaction processing
  • Multi-factor authentication
  • 99.999% uptime for core systems
  • Compliance with banking regulations
