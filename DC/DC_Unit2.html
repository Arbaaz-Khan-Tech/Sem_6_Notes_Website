Unit 2: Communication in Distributed Systems

Understanding communication models and mechanisms

Message Passing Models

Key Concepts

Synchronous Communication

Sender and receiver must be active at the same time. The sender blocks until the message is received and processed.

Asynchronous Communication

Sender doesn't wait for the receiver. Messages are stored in a buffer until the receiver is ready to process them.

Message Buffering

Messages are stored in buffers to handle timing differences between sender and receiver.

Message-Oriented vs Stream-Oriented

Feature Message-Oriented Stream-Oriented
Data Unit Discrete messages Continuous data stream
Timing No timing constraints Real-time delivery required
Ordering Message order may vary Strict ordering required
Use Cases File transfer, email Video streaming, VoIP
Buffer Management Simple message queues Complex buffer management

Remote Procedure Call (RPC)

RPC Components

  • Client Stub: Marshals parameters and sends request
  • Server Stub: Unmarshals parameters and calls procedure
  • RPC Runtime: Handles communication and error handling
  • IDL (Interface Definition Language): Defines procedure interfaces

RPC Steps

  1. Client calls local procedure
  2. Client stub marshals parameters
  3. Message sent to server
  4. Server stub unmarshals parameters
  5. Server executes procedure
  6. Result returned to client

Code Migration

Types of Code Migration

Process Migration

Moving an entire process from one machine to another. Includes process state, memory, and resources.

Object Migration

Moving objects between different address spaces. Requires serialization and deserialization.

Agent Migration

Moving autonomous agents between systems. Includes code, state, and execution context.
