Issues in Distributed Systems

Key Challenges

Heterogeneity

Different hardware, operating systems, and programming languages must work together seamlessly. This requires standardized protocols and interfaces.

Scalability

Systems must handle growing numbers of users and resources without significant performance degradation. This involves efficient resource allocation and load balancing.

Openness

Systems should be extensible and reimplementable. This requires well-defined interfaces and protocols that are publicly available.

Security

Protection against unauthorized access and attacks is crucial. This includes authentication, authorization, and encryption mechanisms.

Failure Handling

Systems must be resilient to partial failures. This involves fault detection, recovery mechanisms, and redundancy.

Concurrency

Multiple users and resources must be managed efficiently. This requires synchronization mechanisms and conflict resolution.

Transparency

Complexity should be hidden from users. This includes location, migration, replication, and failure transparency.

RPC & RMI

Remote Procedure Call (RPC)

  • Allows a program to call a procedure on another machine
  • Transparent to the programmer
  • Uses message passing for communication
  • Handles parameter marshalling
  • Provides error handling

Remote Method Invocation (RMI)

  • Java's implementation of RPC
  • Object-oriented approach
  • Supports distributed objects
  • Provides garbage collection
  • Includes security features

Process vs Thread

Feature Process Thread
Resource Allocation Independent resources Shares resources with parent process
Memory Space Separate memory space Shares memory space
Creation Cost High Low
Communication IPC mechanisms required Direct communication possible
Context Switching Slow Fast
Dependency Independent Dependent on parent process
Failure Impact Doesn't affect other processes Can affect other threads
