Unit 5: Consistency & Replication

Understanding data consistency and replication in distributed systems

Data-centric Consistency Models

Key Models

Consistency Models:
  1. Strong Consistency
    • All replicas are identical
    • Reads always return most recent write
    • High latency, low availability
  2. Sequential Consistency
    • Operations appear to execute in some sequential order
    • All processes see same order
    • Good balance of consistency and performance
  3. Causal Consistency
    • Preserves causally related operations
    • Allows concurrent operations
    • Better performance than sequential
  4. Eventual Consistency
    • Replicas eventually become consistent
    • High availability, low latency
    • May return stale data

Real-life Applications

  • Distributed Databases
    • MongoDB (eventual consistency)
    • Cassandra (tunable consistency)
    • DynamoDB (eventual consistency)
  • Content Delivery Networks
    • Cache consistency
    • Content replication
    • Edge server synchronization

Client-centric Consistency

Types of Client-centric Consistency

Consistency Types:
  1. Read-your-writes
    • Client always sees its own writes
    • Useful for user sessions
    • Example: Social media posts
  2. Monotonic Reads
    • Client never sees older data
    • Ensures forward progress
    • Example: News feeds
  3. Monotonic Writes
    • Writes are applied in order
    • Prevents write conflicts
    • Example: Bank transactions
  4. Writes-follow-reads
    • Writes are based on latest read
    • Maintains causality
    • Example: Collaborative editing

Advantages

  • Better user experience
  • Reduced latency
  • Improved availability
  • Simpler client logic
  • Better scalability

Disadvantages

  • More complex implementation
  • Additional metadata needed
  • Potential for conflicts
  • Higher storage overhead
  • More complex recovery

Fault Tolerance

Key Concepts

Fault Tolerance is the ability of a system to continue operating properly in the event of failures.

Fault Tolerance Mechanisms:
  1. Replication
    • Multiple copies of data
    • Redundant servers
    • Geographic distribution
  2. Checkpointing
    • Periodic state saving
    • Recovery points
    • State restoration
  3. Error Detection
    • Checksums
    • Heartbeats
    • Timeouts
  4. Recovery
    • Rollback
    • Forward recovery
    • State reconstruction

Real-life Applications

  • Cloud Computing
    • VM migration
    • Load balancing
    • Auto-scaling
  • Distributed Storage
    • Data replication
    • Backup systems
    • Disaster recovery
  • High Availability Systems
    • Failover clusters
    • Redundant systems
    • Load balancing

Advantages

  • High availability
  • Data durability
  • System reliability
  • Business continuity
  • Improved user experience

Disadvantages

  • Higher cost
  • Increased complexity
  • Performance overhead
  • More maintenance
  • Resource requirements
