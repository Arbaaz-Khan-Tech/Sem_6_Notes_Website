Cheat Sheet: Cryptography & System Security

Based on Mumbai University Syllabus & Previous Papers

1. Block Ciphers & Modes

ECB (Electronic Codebook)

  • Encrypts each block independently
  • Weakness: Identical plaintext blocks → identical ciphertext

CBC (Cipher Block Chaining)

  • XORs plaintext with previous ciphertext block
  • Requires IV (Initialization Vector)

2. Public Key Cryptography

RSA

Key Generation:

  1. Choose primes (p, q)
  2. Compute n = p × q, φ(n) = (p-1)(q-1)
  3. Choose e s.t. gcd(e, φ(n)) = 1
  4. Compute d = e⁻¹ mod φ(n)

Encryption: C = Mᵉ mod n

Decryption: M = Cᵈ mod n

Diffie-Hellman

Shared Key: K = gᵃᵇ mod p

Steps:

  1. Alice sends A = gᵃ mod p
  2. Bob sends B = gᵇ mod p
  3. Shared key: K = Bᵃ mod p = Aᵇ mod p

3. Hash Functions

MD5

  • 128-bit hash
  • Vulnerable to collisions

SHA-1

  • 160-bit hash
  • Phased out (collisions)

SHA-256

  • 256-bit hash
  • Secure for most uses

4. SSL/TLS Handshake Protocol

  1. ClientHello: Lists supported cipher suites
  2. ServerHello: Chooses cipher suite, sends certificate
  3. Key Exchange: Pre-master secret exchanged (RSA/DH)
  4. Finished: Encrypted handshake verification

5. Digital Signatures & Certificates

RSA Signature

Sign = Mᵈ mod n
Verify using Signᵉ mod n = M

X.509 Certificate Fields

  • Version
  • Serial Number
  • Issuer
  • Validity
  • Subject
  • Public Key
  • Signature

6. Classical Ciphers

Playfair Cipher

  1. Create 5x5 matrix using keyword (remove duplicates, add remaining letters)
  2. Encrypt pairs of letters:
    • Same row → shift right
    • Same column → shift down
    • Rectangle → swap corners

Hill Cipher

  • Encrypt using matrix multiplication: C = K·P mod 26
  • Key matrix must be invertible modulo 26

7. Security Mechanisms

Firewalls

  • Packet-filtering (Network Layer)
  • Stateful Inspection (Transport Layer)
  • Proxy (Application Layer)

Memory Protection

  • Address randomization
  • Access control lists (ACLs)

8. Threats & Attacks

Session Hijacking

Stealing session IDs via MITM or XSS

Phishing Types

  • Spear
  • Whaling
  • Clone
  • Vishing

Clickjacking

Tricking users to click hidden UI elements

9. Database Security Requirements

  • Confidentiality
  • Integrity
  • Availability
  • Access Control
  • Auditing

10. Key Algorithms

Triple DES

  • Uses 3 keys (K1, K2, K3)
  • 3 stages: Encrypt-Decrypt-Encrypt

DES

  • Block size: 64 bits
  • Key size: 56 bits
  • S-Box: Provides confusion
  • Expansion Permutation: Expands 32-bit input to 48 bits

11. Numericals Quick Guide

RSA Example

  • p = 7, q = 13, e = 5
  • n = 91, φ(n) = 72, d = 29 (since 5 × 29 mod 72 = 1)
  • Encrypt M = 10: 10⁵ mod 91 = 82

Diffie-Hellman Example

  • p = 23, g = 5, a = 6, b = 15
  • Shared key: 5⁶×¹⁵ mod 23 = 2

12. Short Notes
